# 房屋租赁管理系统设计方案

## 一、系统角色划分

角色 | 描述
---|---
**租客** | 浏览房源、签约、支付租金、提交维修请求等
**房东** | 发布房源、管理租约、收租、处理维修等
**管理员** | 审核信息、处理纠纷、数据统计、系统管理

---

## 二、用户端功能设计

### （一）通用功能模块（租客/房东共用）

1. **注册与登录**
   - 手机号/邮箱注册，区分“房东”和“租客”角色。
   - 第三方登录（微信/支付宝）。
   - 实名认证（身份证+人脸识别）。
2. **消息中心**
   - 系统通知（合同到期提醒、租金到账提醒）。
   - 租客与房东的站内私信沟通。
3. **个人中心**
   - 修改个人信息、密码。
   - 实名认证状态查看。
   - 绑定/解绑银行卡（用于租金收付）。

### （二）租客专属功能

1. **房源搜索与筛选**
   - 地图找房（集成高德/百度地图API）。
   - 条件筛选：价格区间、户型、面积、位置、配套设施（电梯/停车场）。
   - 收藏房源、对比房源。
2. **在线签约**
   - 电子合同模板（支持自定义条款）。
   - 在线签名（集成e签宝等第三方服务）。
   - 合同状态跟踪（待签署/已生效/已终止）。
3. **租金支付**
   - 按月支付/季度支付（支持支付宝/微信/银联）。
   - 自动扣款提醒。
   - 历史账单查询与下载。
4. **维修服务**
   - 提交维修申请（文字+图片描述）。
   - 查看维修进度（待处理/维修中/已完成）。
   - 对维修服务评分。
5. **评价系统**
   - 对房源和房东进行星级评分。
   - 撰写文字评价（敏感词过滤机制）。

### （三）房东专属功能

1. **房源管理**
   - 发布/编辑房源信息（标题、描述、图片/视频上传）。
   - 设置租金价格、押金规则、最短租期。
   - 房源状态管理（上架/下架/已出租）。
2. **租约管理**
   - 审核租客申请（查看租客信用分）。
   - 生成电子合同。
   - 租约到期自动提醒续约。
3. **财务管理**
   - 租金收入统计（月度/年度报表）。
   - 提现到银行卡（手续费计算）。
   - 逾期未付租金自动催缴。
4. **维修处理**
   - 接收租客维修请求。
   - 指派维修人员（对接第三方维修公司API）。
   - 维修费用记录与分摊设置（如租客承担部分费用）。

---

## 三、管理端功能设计

1. **审核管理**
   - 房源信息审核（防止虚假信息）。
   - 实名认证人工复核。
   - 用户评价内容审核。
2. **纠纷处理**
   - 租客与房东的投诉工单处理。
   - 强制终止违规合同。
   - 黑名单管理（恶意欠租/虚假房源）。
3. **数据统计**
   - 房源出租率热力图。
   - 用户增长趋势分析。
   - 平台交易总额报表。
4. **系统设置**
   - 权限分配（多级管理员角色）。
   - 平台规则配置（手续费比例、违约金计算规则）。
   - 敏感词库管理。

## 四、技术实现方案

### （一）前端
- **租客/房东端**：Uni-app（兼容小程序+H5）
- **管理端**：Vue3 + Element Plus

### （二）后端
- **框架**：Spring Boot + MyBatis Plus
- **架构**：分布式架构（用户服务、订单服务、消息服务独立部署）

### （三）数据库
- **MySQL**：核心业务数据
- **Redis**：缓存高频访问数据（如房源信息）
- **MongoDB**：存储图片/日志等非结构化数据

### （四）第三方服务
- **支付**：支付宝开放平台
- **电子合同**：法大大/腾讯电子签
- **地图**：高德地图API
- **短信**：阿里云短信服务

### （五）安全设计
- **JWT令牌鉴权**
- **敏感数据加密传输**：HTTPS + AES
- **防刷机制**：短信验证码限流

## 五、核心业务流程图示例

### （一）租客租房流程
搜索房源 → 预约看房 → 提交申请 → 签署合同 → 支付押金/租金 → 入住 → 退房结算

### （二）房东收租流程
发布房源 → 审核租客 → 签订合同 → 收取押金 → 每月自动扣款 → 租约结束退押金

## 六、扩展性设计

1. **智能推荐**：根据租客浏览历史推荐相似房源。
2. **信用体系**：对接芝麻信用分评估租客资质。
3. **IoT集成**：智能门锁（生成临时密码）、水电表自动抄表
